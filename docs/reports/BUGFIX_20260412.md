# Bug 修復記錄 — 2026-04-12

測試驅動發現並修復的三個生產 bug，均透過 `tests/test_generate_images_v2.py` 整合測試驗證。

---

## Bug 1：HTML markdown fence 未正確剝離

**位置：** `src/smart_renderer.py` — `_validate_generated_html()`

**症狀：** Smart mode attempt 1 總是失敗並觸發 retry，日誌輸出：
```
Smart layout attempt 1 failed validation: Generated HTML is missing <!DOCTYPE html> or <html> tag. Retrying...
```

**根本原因：** Haiku 回傳的 HTML 用 markdown fence 包裹（` ```html ... ``` `），末尾有空行。原本的 strip 邏輯：
```python
end = len(lines) - 1 if lines[-1].strip().startswith("```") else len(lines)
```
當 `lines[-1]` 是空行時，條件為 False，closing fence 沒有被移除，導致 `<!DOCTYPE html>` 找不到。

**修復：** 從後往前掃描找第一個 ` ``` ` 行：
```python
end = next(
    (i for i in range(len(lines) - 1, 0, -1) if lines[i].strip().startswith("```")),
    len(lines),
)
```

**影響：** 修復前每張圖消耗 2 次 LLM 呼叫（attempt 1 + retry），修復後 1 次即通過，節省約 50% Haiku token。

## Bug 2：`results` 變數名衝突導致 C 帳號 crash

**位置：** `scripts/generate_images_v2.py` — `generate_images()` 內部

**症狀：** 跑完 B 帳號多張 carousel 後，進入 C 帳號時：
```
KeyError: 'C'
⚠️ Carousel failed: 'B'
```

**根本原因：** carousel 多張渲染的內部變數與外層 `results` dict 同名：
```python
results = {  # 外層：帳號統計 dict
    "A": {...}, "B": {...}, "C": {...}
}
...
results = {}  # 內層：slide index → path，覆蓋了外層！
```
當 B 帳號 carousel 執行後，外層 `results` 被替換成 `{1: path, 2: path, ...}`，迴圈繼續時 `results["C"]` 爆 KeyError。

**修復：** 內層改名為 `slide_results`：
```python
slide_results = {}
with concurrent.futures.ThreadPoolExecutor(max_workers=5) as ex:
    for idx, p in ex.map(lambda s: _render(s), slides):
        slide_results[idx] = p
paths = [slide_results[k] for k in sorted(slide_results)]
```

**影響：** 多帳號有任一帳號跑 carousel 多張時，後續帳號必定 crash。C 帳號實際上從未成功渲染過。

## Bug 3：`Content.from_dict` 缺 `output_path` 欄位

**位置：** `src/content.py` — `Content.from_dict()` 的 `valid_fields`

**症狀：** 生圖成功後，從 DB 重新讀出的 Content 物件 `output_path` 永遠是空字串，即使 DB 裡有值。

**根本原因：** `from_dict` 的白名單缺少 `output_path`：
```python
valid_fields = {
    'id', 'account_type', 'status', 'content_type', 'title', 'body',
    'image_path', 'reasoning', ...  # output_path 未列入
}
data = {k: v for k, v in data.items() if k in valid_fields}
```
DB 讀出的 row 含 `output_path`，但被過濾掉，dataclass 維持預設值 `""`。

**修復：** 加入 `output_path` 到白名單：
```python
valid_fields = {
    'id', 'account_type', 'status', 'content_type', 'title', 'body',
    'image_path', 'output_path', 'reasoning', ...
}
```

**影響：** 任何依賴 `content.output_path` 的下游邏輯（elite_review、發布流程）都無法取得圖片路徑。
