# imgGen 產品藍圖 — Cycle 5（最終循環）
**版本範圍**: v2.1 → v3.0
**撰寫日期**: 2026-03-28
**狀態**: 已核准

## 前言

本文件是 imgGen 的最終規劃循環。v2.0 已實作完整的 CLI 介面、四大 AI 供應商、批次處理、浮水印、主題與格式系統。Cycle 5 的目標是讓這個工具從「功能完整」升級為「體驗完整」——讓每一次使用都更快、更聰明、更有創作者掌控感，並在 v3.0 達到可以驕傲地說「完成了」的狀態。

## 版本藍圖

### v2.1 — Smart Fetch（智慧網頁擷取）

**一句話描述**: 用 trafilatura 取代正規表達式 HTML 剝除，讓 `--url` 真正能擷取文章正文。

**主要功能**:
- 整合 trafilatura 作為主要 URL 內容解析器
- 自動偵測並排除廣告、導覽列、頁尾等雜訊區塊
- 保留文章標題、作者、發布日期作為元資料傳入 AI 提示
- 對無法解析的頁面提供明確的錯誤訊息，而非空白輸出
- 對付費牆或需登入頁面顯示警告提示

**複雜度**: S

---

### v2.2 — Dry Run（試跑模式）

**一句話描述**: `--dry-run` 旗標讓使用者只取得 AI 摘要文字，不產生圖片，快速驗證提示品質。

**主要功能**:
- `--dry-run` 輸出 AI 摘要至 stdout，不啟動 Playwright
- 同時顯示估算的 token 用量與費用（適用有計費的供應商）
- 支援搭配 `--batch` 使用，批次預覽所有摘要
- 以結構化 JSON 格式輸出摘要（方便 shell 管道串接）
- 執行時間顯示，讓使用者評估 AI 回應速度

**複雜度**: S

---

### v2.3 — Prompt Styles（提示風格系統）

**一句話描述**: 內建數種 AI 提示風格（如新聞、學術、社群媒體），並支援自訂風格擴充。

**主要功能**:
- `--style` 參數，預設值為 `default`；可選 `news`、`academic`、`social`、`minimal`
- 每種風格控制摘要的語氣、長度、標點習慣
- `~/.imggenrc` 支援定義自訂風格的提示模板
- 風格可與供應商獨立運作（任何 AI 都能套用任何風格）
- `imggen --list-styles` 列出所有可用風格與說明

**複雜度**: M

---

### v2.4 — Preset System（預設參數組合）

**一句話描述**: 將常用的參數組合命名儲存，一個指令取代十個旗標。

**主要功能**:
- `imggen preset save <名稱>` 將目前參數組合存入 `~/.imggenrc`
- `imggen preset load <名稱>` 套用已儲存的預設值
- `--preset <名稱>` 在單次執行中套用預設，可被後續旗標覆蓋
- `imggen preset list` 以表格顯示所有預設及其參數摘要
- `imggen preset delete <名稱>` 移除預設

**複雜度**: M

---

### v2.5 — Clipboard Output（剪貼簿輸出）

**一句話描述**: `--clipboard` 旗標將產生的圖片直接複製到系統剪貼簿，省去手動開檔步驟。

**主要功能**:
- `--clipboard` 在圖片寫入磁碟的同時複製到剪貼簿（macOS / Linux）
- 支援 PNG 與 WebP 格式（剪貼簿以 PNG 為主，自動轉換）
- 可與 `--output` 同時使用（儲存檔案並複製）
- 若系統剪貼簿不支援，顯示明確警告而非靜默失敗
- 批次模式下忽略此旗標並警告（只有單張才合理）

**複雜度**: S

---

### v2.6 — Custom HTML Themes（自訂 HTML 主題）

**一句話描述**: 允許使用者提供自己的 Jinja2 HTML 模板，完全掌控視覺輸出。

**主要功能**:
- `--theme-file PATH` 載入自訂 HTML 模板（Jinja2 格式）
- 文件化可用的模板變數（`title`、`summary`、`brand`、`date` 等）
- 內建模板可被 `imggen theme export <主題名稱>` 匯出作為自訂起點
- 自訂主題可登錄至 `~/.imggenrc` 並以名稱引用
- 模板語法錯誤提供明確的行號與說明

**複雜度**: M

---

### v2.7 — Output History（輸出歷史記錄）

**一句話描述**: 以 SQLite 記錄每次輸出的元資料，支援查詢與重新產生。

**主要功能**:
- 每次成功輸出自動寫入 `~/.imggen_history.db`（SQLite）
- 記錄欄位：時間戳記、來源文字/URL、AI 供應商、主題、格式、輸出路徑
- `imggen history list` 顯示最近 N 筆記錄（可 `--limit` 控制）
- `imggen history replay <ID>` 以相同參數重新產生指定歷史記錄
- `imggen history clear` 清除全部歷史（附確認提示）

**複雜度**: M

---

### v2.8 — Gallery View（本機圖庫檢視）

**一句話描述**: `imggen gallery` 開啟一個本機靜態 HTML 頁面，瀏覽所有歷史輸出圖片。

**主要功能**:
- 從 SQLite 歷史資料庫讀取，產生靜態 HTML 圖庫
- 以縮圖格狀排列，點擊展開原圖與產生參數
- 支援依主題、格式、供應商篩選
- `--open` 旗標直接在預設瀏覽器開啟（預設行為）
- `--export-gallery PATH` 將圖庫匯出為可攜式獨立 HTML 檔案

**複雜度**: M

## 功能選擇矩陣

以下針對本循環中最具槓桿效果的五個候選功能進行評分，作為優先實作順序的依據。

評分標準說明：
- **用戶衝擊 (1-5)**: 使用頻率 × 痛點深度——這個功能解決多真實的問題？
- **可行性 (1-5)**: 技術難度低、依賴少、不破壞現有行為者得高分。
- **WOW 因子 (1-5)**: 第一次用到時的驚喜感、分享欲、「這就是我要的」的程度。

| 功能候選 | 用戶衝擊 | 可行性 | WOW 因子 | 總分 |
|----------|----------|--------|----------|------|
| **Preset System（預設參數組合）** | **5** | **4** | **5** | **100** |
| Smart Fetch（trafilatura）| 4 | 4 | 3 | 48 |
| Custom HTML Themes（自訂模板）| 3 | 3 | 5 | 45 |
| Output History（SQLite 歷史）| 4 | 3 | 4 | 48 |
| Dry Run（試跑模式）| 3 | 5 | 3 | 45 |

> 總分計算方式：用戶衝擊 × 可行性 × WOW 因子

---

### 勝出功能：**Preset System（v2.4）**

**理由**:

Preset System 是這個工具目前最大的摩擦點解法。imgGen 的核心使用場景是「重複性創作工作流程」——同一個創作者每週產出固定風格的圖卡。現在每次都要輸入 `--theme gradient --format story --provider claude --brand-name "我的品牌" --scale 2 --webp`，即使有 shell alias 也是脆弱的解法。

Preset System 讓工具從「強大但繁瑣」變成「強大且流暢」。這是 solo creator 工具的關鍵躍升：當工具開始「記得你的習慣」，它就從一個程式變成一個助手。

Smart Fetch 和 Output History 雖然也很有價值，但它們解決的是特定情境的問題（URL 解析、歷史查詢）。Preset System 解決的是每次使用都會遇到的問題——這才是真正的槓桿點。

WOW 因子也是關鍵：`imggen --preset weekly-ig --url https://...` 這個體驗，第一次用到就會想告訴別人。

## 決策備忘

| 功能 | 決策 | 原因 |
|------|------|------|
| 多用戶支援 | 永不建置 | 定位為 solo 工具，加入多用戶會破壞設計哲學 |
| 雲端同步 | 永不建置 | 本機工具的核心價值是不依賴網路與第三方服務 |
| GUI 介面 | 延後至需求出現 | CLI 工具的使用者不需要 GUI；若需求出現可考慮 TUI |
| 影片輸出 | 不建置 | 超出靜態圖卡的核心定位，複雜度不成比例 |
| API 伺服器模式 | 不建置 | 違反「無 SaaS、無雲端」的核心約束 |
